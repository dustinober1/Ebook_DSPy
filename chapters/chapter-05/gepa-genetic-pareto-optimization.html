GEPA: Genetic-Pareto Optimization

Multi-objective prompt optimization using genetic algorithms and Pareto frontiers to balance accuracy, cost, and other metrics simultaneously.

Introduction

Optimizing prompts often involves trade-offs: a longer prompt might be more accurate but costlier and slower. GEPA (Genetic-Pareto Optimization) addresses this by finding a set of optimal solutions (the Pareto front) rather than a single "best" one.

Key Concepts

  • Genetic Algorithms: Uses evolution-inspired mechanisms (selection, crossover, mutation) to evolve a population of prompts.
  • Pareto Optimization: Optimizes multiple conflicting objectives simultaneously. A solution is Pareto-optimal if no other solution is better in all objectives.
  • Natural Language Reflections: Uses an LLM to critique and improve prompts, guiding the mutation process intelligently.

The GEPA Algorithm

  1. Initialization: Create a diverse population of prompts.
  2. Evaluation: Test each prompt on multiple objectives (e.g., Accuracy, Efficiency).
  3. Selection: Identify non-dominated solutions (Pareto front).
  4. Evolution:
    • Crossover: Combine parts of successful prompts.
    • Mutation: Modify prompts based on LLM feedback/reflections.
  5. Repeat for N generations.
optimizer = GEPAOptimizer(
    objectives=[
        {"name": "accuracy", "direction": "maximize"},
        {"name": "cost", "direction": "minimize"}
    ],
    generations=10
)

Why use GEPA?

Standard optimizers usually optimize a single metric (like accuracy). GEPA gives you choices: do you want the 98% accurate prompt that costs $0.05/run, or the 96% accurate one that costs $0.01/run? This is critical for production deployment.
