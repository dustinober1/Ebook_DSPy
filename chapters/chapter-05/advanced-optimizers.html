Advanced Optimizers

KNNFewShot, COPRO, fine-tuning, and specialized optimization strategies.

~15 min read

üéØ KNNFewShot: Similarity-Based Example Selection

KNNFewShot dynamically selects the most relevant examples based on similarity to the input query:

from dspy.teleprompt import KNNFewShot

class TopicClassifier(dspy.Module):
    def __init__(self):
        super().__init__()
        self.classify = dspy.Predict("text -> topic")
    
    def forward(self, text):
        return self.classify(text=text)

# Create KNNFewShot optimizer
optimizer = KNNFewShot(
    k=5,                    # Number of examples to select
    trainset=trainset,      # Example pool
    vectorizer=None         # Uses default sentence embeddings
)

# Compile
compiled = optimizer.compile(TopicClassifier(), trainset=trainset)
‚úÖ

When to Use

Large datasets, context-sensitive tasks, domain diversity

‚ö°

Advantage

Selects different examples per query for better relevance

üí∞ COPRO: Cost-Aware Prompt Optimization

COPRO (Collaborative Prompt Optimization) balances performance with cost efficiency: 

from dspy.teleprompt import COPRO

# COPRO optimizes prompts while considering token costs
optimizer = COPRO(
    metric=accuracy_metric,
    breadth=10,           # Number of prompt variants to try
    depth=3,              # Refinement iterations
    init_temperature=1.0  # Diversity in generation
)

# Compile with cost awareness
compiled = optimizer.compile(
    program,
    trainset=trainset,
    eval_kwargs={"num_threads": 4}
)
üí°

Use case: When you need to balance accuracy with API costs or token limits.

üîÄ BootstrapFewShotWithRandomSearch

Extends BootstrapFewShot with random search over configurations:

from dspy.teleprompt import BootstrapFewShotWithRandomSearch

optimizer = BootstrapFewShotWithRandomSearch(
    metric=accuracy_metric,
    max_bootstrapped_demos=8,
    max_labeled_demos=4,
    num_candidate_programs=16,  # Try this many configurations
    num_threads=4                # Parallel evaluation
)

# Finds the best configuration through random search
best_program = optimizer.compile(program, trainset=trainset)
üé≤

Random Search

Explores multiple demo configurations to find optimal

Parallelizable

Evaluates configurations in parallel for speed

üîß Fine-Tuning Optimizers

For production systems with high inference volume, fine-tuning small models can reduce costs:

from dspy.teleprompt import BootstrapFinetune

# Fine-tune a small model on bootstrapped examples
optimizer = BootstrapFinetune(
    metric=accuracy_metric,
    target_model="gpt-3.5-turbo",  # Model to fine-tune
    epochs=3
)

# This creates a fine-tuned model
finetuned = optimizer.compile(
    program,
    trainset=trainset,
    teacher_settings=dict(lm=dspy.settings.lm)  # Use powerful LM as teacher
)
‚ö†Ô∏è

Requirements: Fine-tuning requires 1000+ examples and significant compute time.
