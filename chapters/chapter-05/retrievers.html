Retriever Modules

Connect DSPy to vector databases, search engines, and custom retrieval backends.

~15 min read

üîß DSPy Retriever Interface

DSPy provides a standard retriever interface that works with various backends:

import dspy

# Configure a retriever (we'll show specific backends below)
retriever_model = dspy.ColBERTv2(url="http://...")

# Use dspy.Retrieve module
retrieve = dspy.Retrieve(k=5)  # Get top 5 documents

# Call it with a query
results = retrieve(query="What is machine learning?")

# Results format
for passage in results.passages:
    print(passage)
üí°

Key point: dspy.Retrieve is a module that interfaces with a configured retrieval model (RM).

üóÑÔ∏è ChromaDB Integration

ChromaDB is an open-source embedding database, great for local development:

# Install ChromaDB
pip install chromadb
import chromadb
import dspy
from chromadb.utils import embedding_functions

# Create ChromaDB client
chroma_client = chromadb.Client()

# Use OpenAI embeddings (or any other)
openai_ef = embedding_functions.OpenAIEmbeddingFunction(
    api_key="your-api-key",
    model_name="text-embedding-3-small"
)

# Create a collection
collection = chroma_client.create_collection(
    name="my_documents",
    embedding_function=openai_ef
)

# Add documents
documents = [
    "DSPy is a framework for programming language models.",
    "Python is a programming language.",
    "Machine learning uses data to make predictions.",
    "Neural networks are inspired by the brain.",
    "Transformers revolutionized NLP.",
]

collection.add(
    documents=documents,
    ids=[f"doc_{i}" for i in range(len(documents))]
)

# Create custom retriever function
def chromadb_retrieve(query: str, k: int = 3):
    results = collection.query(
        query_texts=[query],
        n_results=k
    )
    return results["documents"][0]

# Use in a DSPy module
class ChromaRAG(dspy.Module):
    def __init__(self, k=3):
        self.k = k
        self.generate = dspy.ChainOfThought("context, question -> answer")
    
    def forward(self, question):
        # Retrieve
        docs = chromadb_retrieve(question, self.k)
        context = "\n".join(docs)
        
        # Generate
        return self.generate(context=context, question=question)

# Test it
lm = dspy.LM("openai/gpt-4o-mini")
dspy.configure(lm=lm)

rag = ChromaRAG()
result = rag(question="What is DSPy?")
print(result.answer)

üî® Creating a Custom Retriever

You can create a retriever that works with any backend:

import dspy
from typing import List, Optional

class CustomRetriever(dspy.Retrieve):
    """Custom retriever that wraps your search backend."""
    
    def __init__(self, search_client, k: int = 3):
        super().__init__(k=k)
        self.client = search_client
    
    def forward(self, query: str, k: Optional[int] = None) -> dspy.Prediction:
        k = k if k is not None else self.k
        
        # Call your backend
        results = self.client.search(query, num_results=k)
        
        # Return in DSPy format
        return dspy.Prediction(
            passages=[r["text"] for r in results]
        )

üìã Simple In-Memory Retriever

For learning and testing, here's a simple TF-IDF based retriever:

from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity
import numpy as np

class SimpleRetriever:
    """A simple TF-IDF based retriever for testing."""
    
    def __init__(self, documents: list):
        self.documents = documents
        self.vectorizer = TfidfVectorizer()
        self.doc_vectors = self.vectorizer.fit_transform(documents)
    
    def __call__(self, query: str, k: int = 3) -> list:
        # Vectorize query
        query_vector = self.vectorizer.transform([query])
        
        # Calculate similarities
        similarities = cosine_similarity(query_vector, self.doc_vectors)[0]
        
        # Get top k indices
        top_indices = np.argsort(similarities)[::-1][:k]
        
        # Return documents
        return [self.documents[i] for i in top_indices]

# Usage
docs = [
    "DSPy is a framework for programming language models.",
    "Python is widely used in data science.",
    "Neural networks can learn complex patterns.",
    "GPT models are large language models.",
    "Machine learning automates analytical model building.",
]

retriever = SimpleRetriever(docs)
results = retriever("What is DSPy?", k=2)
print(results)
# ['DSPy is a framework for programming language models.', 
#  'GPT models are large language models.']

üéõÔ∏è Key Retrieval Parameters

üî¢

k (number of results)

How many documents to retrieve. Too few = missing info, too many = noise.

üìä

Similarity threshold

Minimum similarity score to include a document. Filters out weak matches.

üìè

Chunk size

How large each document chunk should be. Balance between context and specificity.

üîÄ

Chunk overlap

Overlap between chunks to avoid cutting off important context.

üí°

Starting values: k=5, chunk_size=500 characters, chunk_overlap=50 characters. Tune based on your data.

üìù Key Takeaways

dspy.Retrieve is the standard retriever interface

Multiple backends: ChromaDB, Pinecone, Weaviate, custom

Easy to create custom retrievers for your backend

Key parameter: k controls how many documents to retrieve
