Advanced Optimizers

Beyond basic few-shot optimization‚Äîinstruction tuning, fine-tuning, and more.

~15 min read

üóÇÔ∏è Optimizer Overview

Optimizer What It Optimizes Best For
BootstrapFewShot Few-shot examples Quick improvements
BootstrapFewShotWithRandomSearch Few-shot examples (with search) Better demo selection
MIPRO Instructions + examples Maximum prompt improvement
COPRO Instructions only Better task descriptions
BootstrapFinetune Model weights Production deployment

üéØ MIPRO (Multi-prompt Instruction PRoposal Optimizer)

MIPRO is one of DSPy's most powerful optimizers. It optimizes both instructions and demonstrations:

from dspy.teleprompt import MIPROv2

# Create MIPRO optimizer
optimizer = MIPROv2(
    metric=my_metric,
    num_candidates=10,      # Instruction candidates to try
    init_temperature=1.0,   # Creativity for generating instructions
    verbose=True
)

# Compile
optimized_module = optimizer.compile(
    my_module,
    trainset=trainset,
    max_bootstrapped_demos=3,
    max_labeled_demos=3,
    num_trials=15,          # Optimization budget
    minibatch_size=25,
    minibatch_full_eval_steps=10,
    requires_permission_to_run=False
)
üìù

What MIPRO Does

Generates multiple instruction candidates, evaluates them, and selects the best.

‚ö°

When to Use

When you've maxed out improvements from BootstrapFewShot and want more gains.

‚ö†Ô∏è

Cost Warning

MIPRO uses many LM calls. Start with small num_trials.

üìù COPRO (Collaborative Prompt Optimization)

COPRO focuses on optimizing the instruction/prompt text itself:

from dspy.teleprompt import COPRO

optimizer = COPRO(
    metric=my_metric,
    breadth=10,     # Number of instruction variants
    depth=3,        # Iterations of refinement
    init_temperature=1.4
)

optimized_module = optimizer.compile(
    my_module,
    trainset=trainset,
    eval_kwargs={"num_threads": 4}
)
üí°

COPRO vs MIPRO: COPRO is simpler and faster, focusing only on instructions. MIPRO is more comprehensive but more expensive.

üîß BootstrapFinetune

For production systems, fine-tune a smaller model to match the quality of a larger one:

from dspy.teleprompt import BootstrapFinetune

# Configure a teacher model (large, expensive)
teacher = dspy.LM("openai/gpt-4o")

# Configure a student model (small, cheap, to be fine-tuned)
student = dspy.LM("openai/gpt-3.5-turbo")

optimizer = BootstrapFinetune(
    metric=my_metric
)

# The optimizer will:
# 1. Use teacher to generate successful examples
# 2. Fine-tune student on those examples
# 3. Return a module using the fine-tuned student

optimized_module = optimizer.compile(
    my_module,
    trainset=trainset,
    teacher_settings={"lm": teacher},
    student_settings={"lm": student}
)
üí∞

Cost Benefit

One-time fine-tuning cost ‚Üí cheaper inference forever.

üöÄ

Performance

Fine-tuned small models can match larger models on specific tasks.

‚öôÔ∏è

Requirements

Requires access to fine-tuning API (OpenAI, Anyscale, etc.).

üéØ Choosing the Right Optimizer

üìä

Small dataset (<20 examples)

Use BootstrapFewShot - simple and effective.

üìà

Medium dataset (20-100 examples)

Use BootstrapFewShotWithRandomSearch for better selection.

üéØ

Need better instructions

Use COPRO to optimize task descriptions.

‚ö°

Maximum improvement needed

Use MIPRO for comprehensive optimization.

üè≠

Production deployment

Use BootstrapFinetune for cost-efficient inference.

üîó Combining Optimizers

You can chain optimizers for maximum improvement:

# Step 1: Start with basic bootstrap
bootstrap = dspy.BootstrapFewShot(metric=my_metric)
bootstrapped_module = bootstrap.compile(my_module, trainset=trainset)

# Step 2: Optimize instructions with COPRO
copro = COPRO(metric=my_metric, breadth=5, depth=2)
copro_optimized = copro.compile(
    bootstrapped_module, 
    trainset=trainset
)

# Step 3: Final refinement with random search
final_optimizer = BootstrapFewShotWithRandomSearch(
    metric=my_metric,
    num_candidate_programs=10
)
final_module = final_optimizer.compile(
    copro_optimized,
    trainset=trainset,
    valset=devset
)

# Evaluate the progression
print(f"Baseline: {evaluate(my_module):.2%}")
print(f"After Bootstrap: {evaluate(bootstrapped_module):.2%}")
print(f"After COPRO: {evaluate(copro_optimized):.2%}")
print(f"After Final: {evaluate(final_module):.2%}")

üìù Key Takeaways

MIPRO optimizes both instructions and examples.

COPRO focuses on improving instruction text.

BootstrapFinetune creates cheaper production models.

Chain optimizers for incremental improvement.

Start simple and add complexity only if needed.
